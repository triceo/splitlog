Messaging

Splitlog seems to be one line behind the watched log. What's wrong?
Well, nothing is. In order to be able to tell you where log messages end, we first need to see the beginning of another message. We cannot predict the future. Therefore, a message will only be received when another one has already begun. Please refer to the documentation on Message Lifecycle and Splitting to learn more.
